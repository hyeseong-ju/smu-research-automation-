# SMU Research Automation Tool

**Status: In Development**

## The Problem
At SMU's Office of Research Strategy, the grant and funding 
opportunity compilation process is entirely manual. Student assistants must collect and research data 
scattered across international databases and organize the data completely manually. This brings the scalability, efficiency, and data management issues. 

## What I'm Building
A web application to automate this pipeline:
- Automated sourcing across global funding databases (potential API usage: Perplexity)
- Structured data cleaning and standardisation
- RAG model with SMU existing database 
- Routing curated opportunities to relevant faculty (Email drafts, real-time notifications)
- Replacing a multi-hour manual workflow with a 
  repeatable, scalable system

## Why
After enrolling into this role, I mapped the workflow within days
and identified that the manual process was the core 
bottleneck for the current early phase for this program SMU office research and strategy has launched. Seeing these issues, I am building the fix without being asked to. I hope to bring a more efficient and replicable workflow to SMU's office. 

## Stack (planned)
Python · Web Scraping · Data Pipeline · Automation
